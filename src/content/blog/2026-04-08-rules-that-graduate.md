---
title: "rules that graduate"
description: "lessons start in one project. the good ones become global."
pubDate: 2026-04-08
---

a rule starts because something went wrong in a specific project. 47 sessions jumping to code without thinking — so that project gets a rule about stating assumptions first.

a few weeks later, the same pattern shows up in a different project. the rule gets cited. then cited again. at some point it stops being a project rule and becomes a global one.

that's the graduation path: project lesson → cited across sessions → promoted to global behavioral standard.

the knowledge base tracks this naturally. cite_count goes up as more sessions reference a lesson. when a lesson hits enough citations across different projects, it's clearly not project-specific anymore. it moves to the global rules.

right now there are about 80 global documents. most started as one project's fix for one project's problem. the system surfaces what's universal without me having to decide upfront.

the opposite path exists too. some rules that seemed universal turned out to be project-specific. they got cited in one context but never another. those stay where they are. no harm in it — a rule that helps one project is still a rule that helps.
