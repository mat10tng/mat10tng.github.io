<div slot="insight">
## Principle
Prompting is a fallback. Reliability comes from the **system** surrounding the model. 

## The Problem
"Prompt Debt": Context window saturation caused by appending competing instructions (e.g., "be concise," "follow X") when a model fails.
</div>
</Section>

<Section>
<div slot="insight">
## Implementation: Bootloader
Establish boundaries in a project root file (e.g., `CLAUDE.md`, `.cursorrules`) to enforce retrieval over memory.
</div>

<div slot="technical">
```markdown
# Project Standards
- **Role:** Human Architect directs; Worker executes.
- **Rules:** State approach before code. Wait for confirmation.
- **Knowledge:** Search local file system before implementing.
```
</div>
</Section>

<Section>
<div slot="insight">
## The Sync Skill
Force a context refresh to synchronize the model with active technical constraints.
</div>

<div slot="technical">
**Command:** `/sync`
**Logic:** "Review structure, read latest logs, verify constraints. Do not proceed until verified."
</div>
</Section>

<Section>
<div slot="insight">
## Result
Short prompts, complex execution. The architecture handles the weight, allowing the model to focus on generation.
</div>
</Section>
