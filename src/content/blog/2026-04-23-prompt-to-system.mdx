<div slot="insight">
## Principle
Prompting is a fallback. Reliability is achieved by moving from adjective-heavy instructions to a persistent **system** that surrounds the model.

## The Memory Structure
A system requires two distinct layers of memory to function:
1. **Sessions**: Short-term logs of activity (what was tried, what failed).
2. **Lessons**: Long-term engineering standards (distilled logic).
</div>

<div slot="technical">
```text
~/memory/
├── global/
│   └── lessons/ (universal standards)
└── project-x/
    ├── sessions/ (activity logs)
    └── lessons/ (project-specific logic)
```
</div>
</Section>

<Section>
<div slot="insight">
## Implementation: Bootloader
Define the system's operating environment in a project root file (e.g., `CLAUDE.md`, `.cursorrules`). This enforces retrieval over memory and establishes the architect/LLM relationship.
</div>

<div slot="technical">
```markdown
# Project Standards
- **Role:** Human Architect directs; LLM executes.
- **Recall:** Always search `~/memory/` for existing logic before work.
- **Log:** After each task, record the technical outcome to `sessions/`.
- **Distill:** If a new reusable rule is identified, propose it for `lessons/`.
```
</div>
</Section>

<Section>
<div slot="insight">
## Creation of Lessons
The system initializes with a simple rule: **Capture first, distill second.** 

When the LLM identifies a recurring pattern or a permanent fix, it creates a new `.md` file in the `lessons/` directory. This turns a one-time conversation into a permanent system standard.
</div>

<div slot="technical">
**Manual Creation Skill:**
> "Review our last three changes. Propose a new entry for `project-x/lessons/` that would have prevented any friction we encountered today."
</div>
</Section>

<Section>
<div slot="insight">
## Context Synchronization
A context refresh skill ensures the LLM synchronizes with the system's current state and previously documented logic before beginning work.
</div>

<div slot="technical">
**Command:** `/sync`
**Logic:** "Read structure, latest session logs, and distilled lessons. Verify constraints. Do not proceed until verified."
</div>
</Section>
