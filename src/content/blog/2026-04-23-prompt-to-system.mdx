<div slot="insight">
## Principle
Reliability is achieved by moving from adjective-heavy prompts to a persistent **system** that surrounds the model.

## The Memory Structure
A system requires two distinct layers of memory to function:
1. **Sessions**: Short-term logs of activity (what was tried, what failed).
2. **Lessons**: Long-term engineering standards (distilled logic).
</div>

<div slot="technical">
```text
[root]/memory/
├── global/
│   └── lessons/
└── project-x/
    ├── sessions/
    └── lessons/
```
</div>
</Section>

<Section>
<div slot="insight">
## Implementation: Bootloader
The Bootloader defines the environment inside the **LLM**. A root file establishes the architect/LLM relationship and forces retrieval over model memory.
</div>

<div slot="technical">
**Example: `CLAUDE.md` or `.cursorrules`**
```markdown
# Project Standards
- **Role:** Architect directs; LLM executes.
- **Recall:** Always search `[root]/memory/` for logic.
- **Log:** Record outcome to `sessions/`.
- **Distill:** Propose reusable rules for `lessons/`.
```
</div>
</Section>

<Section>
<div slot="insight">
## The Self-Curation Loop
A static system degrades. Continuous improvement is achieved through a mechanism that graduates local fixes into universal standards.

1. **Detection**: Identify recurring friction.
2. **Local Fix**: Add a rule to project `lessons/`.
3. **Graduation**: Move the logic to `global/lessons/`.
</div>

<div slot="technical">
**Distillation Skill:**
> "Review last 7 days of logs. Identify 3 recurring patterns. Move to `global/lessons/`. Update project bootloader."
</div>
</Section>

<Section>
<div slot="insight">
## Context Synchronization
A context refresh skill ensures the LLM synchronizes with the system's current state and distilled logic before beginning work.
</div>

<div slot="technical">
**Command:** `/sync`
**Logic:** "Read structure, latest session logs, and distilled lessons. Verify constraints. Do not proceed until verified."
</div>
</Section>
