<Section>
<div slot="insight">
## Principle
Prompting is a fallback. Reliability is achieved by moving from adjective-heavy instructions to a persistent **system** that surrounds the model.

## The Setup
A system requires a structured environment. This removes reliance on model memory and ensures context persists across independent work sessions.
</div>

<div slot="technical">
```text
~/vault/
├── global/
│   └── lessons/
└── project-x/
    ├── sessions/
    └── lessons/
```
</div>
</Section>

<Section>
<div slot="insight">
## Implementation: Bootloader
Define the system's operating environment in a project root file (e.g., `CLAUDE.md`, `.cursorrules`). This enforces retrieval over memory and establishes the architect/worker relationship.
</div>

<div slot="technical">
```markdown
# Project Standards
- **Role:** Human Architect directs; Worker executes.
- **Log:** Record outcome to `~/vault/project-x/sessions/[date]-[goal].md`.
- **Knowledge:** Search local file system before implementing.
```
</div>
</Section>

<Section>
<div slot="insight">
## Context Synchronization
A context refresh skill ensures the model synchronizes with the system's current state and previously documented logic before beginning work.
</div>

<div slot="technical">
**Command:** `/sync`
**Logic:** "Read structure, latest session logs, and distilled lessons. Verify constraints. Do not proceed until verified."
</div>
</Section>

<Section>
<div slot="insight">
## Result
The shift from a prompt to a system. The model inherits its environment from the vault automatically, reducing the overhead of manual instruction.
</div>
</Section>
