## Abstract
Automation removes manual bookkeeping. By persisting system metadata in a root `state.json` file, the architecture can autonomously trigger maintenance tasks—such as distillation and health checks—without direct intervention from the Architect.

## Core Metrics
Measurement provides the data necessary to verify that architectural changes actually improve model performance:
- **Reuse Rate**: Frequency of sessions utilizing distilled lessons.
- **Success Rate**: Frequency of tasks completed without technical regression.
- **Rule Count**: Growth of distilled engineering standards in Memory.
</div>

<div slot="technical">
**Global State (`memory/state.json`)**
```json
{
  "last_distill_date": "2026-04-17",
  "sessions_since_cleanup": 12,
  "system_health_score": 0.94
}
```

## Autonomous Maintenance
Interface (MCP) tools use the root state data to determine operational schedules. If state criteria are met, the model executes curation logic before accepting project goals.
</div>

<div slot="technical">
**Automation Logic:**
```javascript
// Check inside a tool call
const state = await fs.readFile("memory/state.json");
if (daysSince(state.last_distill) > 7) {
    await runDistillation();
    await updateState({ last_distill: now() });
}
```
</div>
</Section>

<Section>
<div slot="insight">
## The Automated Loop
The result is a self-maintaining system. It uses its own data to evolve, ensuring that engineering standards compound over time without manual context management.

1.  **Boot**: LLM reads root state and project rules.
2.  **Maintenance**: LLM runs global curation autonomously.
3.  **Execute**: Technical work proceeds with fresh standards.
4.  **Persist**: Session data and updated state are saved to Memory.
</div>
