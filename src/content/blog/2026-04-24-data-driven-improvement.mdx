## The Answer
**Automating context management via the Model Context Protocol (MCP) ensures that your engineering standards compound over time without manual intervention.**

By persisting system metadata in a root `state.json` file, you enable your architecture to perform its own maintenance—including scaffolding new projects and logging outcomes—autonomously based on real-time success metrics.
</div>
</Section>

<Section>
<div slot="insight">
## 1. System Health Metrics
Measurement provides the evidence needed to verify that architectural changes actually improve model performance.

- **Reuse Rate**: How often the model utilizes distilled lessons.
- **Success Rate**: Frequency of tasks completed without technical regression.
- **Maintenance**: Auto-scaffolding and auto-distillation success rates.
</div>

```json
}
```

<div slot="insight">
## 2. Autonomous Maintenance
The Interface (MCP) tools use state data to determine when to run. If a project directory is missing, or the state is stale, the model executes the maintenance logic before accepting architectural goals.
</div>

<div slot="technical">
**Bridge Tool Logic:**
```javascript
// Auto-Init logic inside a tool call
const projectPath = `memory/${currentProject}`;
if (!await fs.exists(projectPath)) {
    await runAutoScaffold(currentProject);
}

// Auto-Distill logic
if (daysSince(state.last_distill) > 7) {
    await runDistillation();
}
```
</div>
</Section>

<Section>
<div slot="insight">
## The Automated Workflow
1. **Boot**: Model reads global state and auto-initializes project if missing.
2. **Maintenance**: Model triggers curation/cleanup autonomously.
3. **Execute**: Model works with fresh standards.
4. **Persist**: Model saves session logs and updated state automatically.
</div>
