## Principle
Automation removes human bookkeeping. By persisting system metadata in `state.json`, the architecture can autonomously trigger maintenance tasks like distillation and health checks.
</div>

<div slot="technical">
**State Data (`state.json`):**
```json
{
  "last_distill_date": "2026-04-17",
  "sessions_since_cleanup": 12,
  "system_health_score": 0.94
}
```
</div>
</Section>

<Section>
<div slot="insight">
## Autonomous Maintenance
The Interface (MCP) tools use the state data to decide when to run. If the `last_distill_date` is too old, the model executes the curation logic before accepting new work from the Architect.
</div>

<div slot="technical">
**Bridge Tool Logic:**
```javascript
// Example check inside a tool call
const state = await readState();
if (daysSince(state.last_distill) > 7) {
    await runDistillation();
    await updateState({ last_distill: now() });
}
```
</div>
</Section>

<Section>
<div slot="insight">
## The Automated Workflow
1.  **Boot**: LLM reads `state.json`.
2.  **Maintenance**: If state criteria are met, LLM runs curation/cleanup autonomously.
3.  **Execute**: Technical work proceeds with fresh standards.
4.  **Persist**: Session and new state metadata are saved to Memory.
</div>
</Section>

<Section>
<div slot="insight">
## Result
A data-driven, self-running architecture. The system maintains itself based on its own internal state, ensuring that engineering standards compound without manual Architect intervention.
</div>
</Section>
