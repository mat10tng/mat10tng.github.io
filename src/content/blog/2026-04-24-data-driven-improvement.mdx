## Principle
Automation removes human bookkeeping. By persisting metadata in a root `state.json`, the architecture can autonomously trigger maintenance tasks like distillation and health checks.
</div>

<div slot="technical">
**Global State (`memory/state.json`):**
```json
{
  "last_distill_date": "2026-04-17",
  "sessions_since_cleanup": 12,
  "system_health_score": 0.94
}
```
</div>
</Section>

<Section>
<div slot="insight">
## Autonomous Maintenance
The Interface (MCP) tools use the root state data to decide when to run. If the state is stale, the model executes curation logic before accepting new project work.
</div>

<div slot="technical">
**Interface Tool Logic:**
```javascript
// Check inside a tool call
const state = await fs.readFile("memory/state.json");
if (daysSince(state.last_distill) > 7) {
    await runDistillation();
    await updateState({ last_distill: now() });
}
```
</div>
</Section>

<Section>
<div slot="insight">
## The Automated Workflow
The result is a system that maintains itself based on its highest-level internal state, ensuring engineering standards compound without manual intervention.

1.  **Boot**: LLM reads root state and project system rules.
2.  **Maintenance**: LLM runs global curation autonomously.
3.  **Execute**: Technical work proceeds with fresh standards.
4.  **Persist**: Session data and updated state are saved to Memory.
</div>
</Section>
