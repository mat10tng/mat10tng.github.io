**ELI5:** Now that we have a memory system, we use automation (MCP) so we don't have to manually tell the AI to save or read files. We also track numbers to make sure the system is actually getting better over time.

## Principle
Automation removes the boring bookkeeping. By saving system info in a `state.json` file, the architecture can trigger its own maintenance—like cleaning up old logs and updating rules—without you asking.
</div>

<div slot="technical">
**System State (`memory/state.json`)**
```json
{
  "last_distill_date": "2026-04-17",
  "sessions_since_cleanup": 12,
  "system_health_score": 0.94
}
```
</div>
</Section>

<Section>
<div slot="insight">
## Automatic Maintenance
The Interface (MCP) tools check the state file to decide when to run. If the system is getting "stale," the model handles the cleanup before starting new project work.
</div>

<div slot="technical">
**Automation Logic:**
```javascript
// Check inside a tool call
const state = await fs.readFile("memory/state.json");
if (daysSince(state.last_distill) > 7) {
    await runDistillation();
    await updateState({ last_distill: now() });
}
```
</div>
</Section>

<Section>
<div slot="insight">
## The Automated Loop
The result is a system that maintains itself. It uses its own data to get smarter, ensuring engineering standards grow without you having to manage folders manually.

1.  **Boot**: The model reads your state and rules.
2.  **Fix**: The model cleans itself up if needed.
3.  **Work**: You build, and the model follows the latest rules.
4.  **Save**: The model saves its own logs automatically.
</div>
</Section>
