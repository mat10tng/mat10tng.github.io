<div slot="insight">
## Principle
Automation removes human bookkeeping. By persisting system metadata in a root `state.json`, the architecture can autonomously trigger maintenance tasks like distillation and health checks across the entire Memory.
</div>

<div slot="technical">
**Global State (`memory/state.json`):**
```json
{
  "last_distill_date": "2026-04-17",
  "sessions_since_cleanup": 12,
  "system_health_score": 0.94
}
```
</div>
</Section>

<Section>
<div slot="insight">
## Autonomous Maintenance
The Interface (MCP) tools use the root state data to decide when to run. If the `last_distill_date` in `state.json` is too old, the model executes the curation logic before accepting new project work.
</div>

<div slot="technical">
**Interface Tool Logic:**
```javascript
// Check inside a tool call
const state = await fs.readFile("memory/state.json");
if (daysSince(state.last_distill) > 7) {
    await runDistillation();
    await updateState({ last_distill: now() });
}
```
</div>
</Section>

<Section>
<div slot="insight">
## The Automated Workflow
1.  **Boot**: LLM reads root `state.json` and project **System Rules**.
2.  **Maintenance**: If state criteria are met, LLM runs global curation autonomously.
3.  **Execute**: Technical work proceeds with fresh standards.
4.  **Persist**: Session data and updated root state are saved to Memory.
</div>
</Section>

<Section>
<div slot="insight">
## Result
A data-driven, self-running architecture. The system maintains itself based on its own internal state, ensuring that engineering standards compound without manual Architect intervention.
</div>
</Section>
