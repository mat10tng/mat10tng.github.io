## The Answer
**Automating context management via the Model Context Protocol (MCP) ensures that your engineering standards compound over time without manual intervention.**

By persisting system metadata in a root `state.json` file, you enable your architecture to evolve autonomously based on real-time success metrics, removing the manual labor of bookkeeping.
</div>
</Section>

<Section>
<div slot="insight">
## 1. System Health Metrics
Measurement provides the evidence needed to verify that architectural changes actually improve model performance.

- **Reuse Rate**: How often the model utilizes distilled lessons.
- **Success Rate**: Frequency of tasks completed without technical regression.
- **Drift**: Measurable deviation from the established standards.
</div>

<div slot="technical">
**Global State (`memory/state.json`)**
```json
{
  "last_distill_date": "2026-04-17",
  "sessions_since_cleanup": 12,
  "system_health_score": 0.94
}
```
</div>
</Section>

<Section>
<div slot="insight">
## 2. Autonomous Triggers
Instead of manual commands, the Interface (MCP) uses state data to determine operational schedules. If criteria are met, the model handles the cleanup *before* starting new work.
</div>

<div slot="technical">
**Interface Tool Logic:**
```javascript
// Check inside a tool call
const state = await fs.readFile("memory/state.json");
if (daysSince(state.last_distill) > 7) {
    await runDistillation();
    await updateState({ last_distill: now() });
}
```
</div>
</Section>

<Section>
<div slot="insight">
## The Automated Workflow
1. **Boot**: Model reads global state.
2. **Maintenance**: Model triggers curation if stale.
3. **Execute**: Model works with fresh standards.
4. **Persist**: Model saves its own logs automatically.
</div>
