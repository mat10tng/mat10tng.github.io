## Principle
Automation removes the human bookkeeping required to maintain a self-curating system. Measurement provides the data necessary to verify that architectural changes actually improve performance.
</div>

<div slot="technical">
**Key Metrics:**
- **Reuse Rate**: % of sessions utilizing previous lessons.
- **Success Rate**: % of tasks completed without regression.
- **Rule Count**: Growth of distilled standards.
</div>
</Section>

<Section>
<div slot="insight">
## The Automated Workflow
How to use the system when the Interface (MCP) is active:

1.  **Initialize**: The model automatically executes `read_lessons` via MCP tools at startup.
2.  **Execute**: Technical work proceeds. The LLM has direct access to standards without manual prompting.
3.  **Persist**: On task completion, the model calls `save_session` autonomously.
</div>

<div slot="technical">
**Example Interaction:**
> Architect: "Build the auth login flow."
>
> LLM: *Calls `read_lessons`*
> LLM: *Implements flow*
> LLM: *Calls `save_session`*
>
> LLM: "Done. Session logged to Memory."
</div>
</Section>

<Section>
<div slot="insight">
## Self-Running Automation (MCP)
The Model Context Protocol (MCP) enables the system to become autonomous. The manual skills from Step 01 are converted into deterministic tools.
</div>

<div slot="technical">
**Bridge Tools:**
- **`save_session`**: Autonomous logging.
- **`read_lessons`**: Autonomous retrieval.
- **`distill_lessons`**: Autonomous curation.
</div>
</Section>

<Section>
<div slot="insight">
## Scheduled Distillation
To make the system "self-running," the curation logic is scheduled. A background task triggers a review of logs to identify new standards.
</div>

<div slot="technical">
**Scheduled Task Logic:**
> "Triggered at 00:00: Read all sessions from yesterday. Extract recurring fixes. Propose graduation to `global/lessons/`."
</div>
</Section>

<Section>
<div slot="insight">
## Result
A data-driven, self-running architecture. The system maintains itself autonomously, with every architectural shift verified by measurable success rates.
</div>
</Section>
