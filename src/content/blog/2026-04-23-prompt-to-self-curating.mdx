## The Answer
**You can eliminate repetitive prompting and force your technical knowledge to compound across your entire portfolio by establishing an autonomous, project-agnostic memory system.**

Reliability comes from an architecture that performs maintenance by default. By defining structural rules once, you ensure that every session—on any project—initializes, curates, and persists its own engineering standards without manual intervention.
</div>
</Section>

<Section>
<div slot="insight">
## 1. Portfolio Architecture
Compounding requires a strict directory hierarchy. This ensures the model has a deterministic location to store and retrieve history for any project in your portfolio.
</div>

<div slot="technical">
**Unified Memory Structure**
```text
memory/
├── state.json (global health)
├── global/
│   └── lessons/ (portfolio standards)
└── project-x/
    ├── sessions/ (daily logs)
    └── lessons/ (project fixes)
```

**Setup (Linux / macOS)**
```bash
mkdir -p memory/global/lessons
```
</div>
</Section>

<Section>
<div slot="insight">
## 2. System Rules: The Auto-Run Logic
The system is directed by **System Rules** (e.g. `CLAUDE.md`, `.cursorrules`). These are not just instructions; they are the execution logic that forces the model to maintain the architecture autonomously.
</div>

<div slot="technical">
**Mandatory Bootloader Logic**
```markdown
# Project Standards
- **Initialization:** On boot, identify current project. If `memory/[project-name]` does not exist, run `/init-project` immediately.
- **Maintenance:** Check `memory/state.json`. If `last_distill` > 7 days, execute `/distill` before accepting goals.
- **Persistence:** On session completion, autonomously execute `/end-session`.
- **Role:** Architect directs; LLM executes and maintains.
```
</div>
</Section>

<Section>
<div slot="insight">
## 3. Autonomous Skills
The model uses three core skills to maintain the loop without being prompted.

**Initialize**: Scaffolds the environment for new projects.
**Capture**: Persists technical outcomes at the end of work.
**Graduation**: Identifies rules with portfolio-wide utility.
</div>

<div slot="technical">
**Skill: `/init-project`**
> "Scaffold the memory structure: `memory/[project-name]/sessions/` and `memory/[project-name]/lessons/`."

**Skill: `/end-session`**
> "Summarize technical decisions. Save to `memory/[project-name]/sessions/[date]-[goal].md`."

**Skill: `/distill`**
> "Review logs. Identify rules to graduate to `global/lessons/`. Update `memory/state.json`."
</div>
</Section>

<Section>
<div slot="insight">
## The System-Led Workflow
Execution follows a deterministic, autonomous cycle:
1. **Boot**: LLM scaffolds or synchronizes project context.
2. **Curation**: LLM graduates rules if the global state is stale.
3. **Execute**: Architect directs; LLM implements.
4. **Persist**: LLM logs the session automatically before exit.
</div>
</Section>

## Result
A self-curating portfolio. The LLM inherits its environment and maintains its own infrastructure automatically, ensuring your engineering standards grow across every project you touch.
