<Section>
<div slot="insight">
## Principle
Prompting is a fallback. Reliability is achieved by moving from adjective-heavy instructions to a persistent **system** that surrounds the model. 

## The Requirement
A system requires a structured environment for context to persist across independent work sessions. This is implemented as a **Memory** directory on the local file system.
</div>
</Section>

<Section>
<div slot="insight">
## 1. Memory Structure
Memory is divided into two distinct layers:
- **Sessions**: Short-term logs of activity (what was tried, what failed).
- **Lessons**: Long-term engineering standards (distilled logic).
</div>

<div slot="technical">
```text
memory/
├── global/
│   └── lessons/ (universal)
└── project-x/
    ├── sessions/ (logs)
    └── lessons/ (local rules)
```
</div>
</Section>

<Section>
<div slot="insight">
## 2. The Bootloader
The Bootloader defines the operating environment inside the **LLM**. A root file (e.g., `CLAUDE.md`, `.cursorrules`) establishes the architect/LLM relationship and providing the model with its "save" and "distill" skills.
</div>

<div slot="technical">
**Bootloader (Project Root)**
<div class="bootloader-snippet">
```markdown
# Project Standards
- **Role:** Architect directs; LLM executes.
- **Recall:** Search `memory/` for logic.
- **Rules:** Use `/end-session` to log work.
- **Skills:** [/end-session, /distill]
```
</div>
</div>
</Section>

<Section>
<div slot="insight">
## 3. The Daily Workflow
How to use the system in a standard work session:

1.  **Sync**: Start the session by running `/sync`. The LLM reads the vault and bootloader rules.
2.  **Architect**: Provide the technical goal. The LLM implements it using the existing lessons.
3.  **Capture**: Before closing, run `/end-session`. The LLM summarizes and saves the log.
</div>

<div slot="technical">
**Step 01: The `/sync` Skill**
> "Review current structure and lessons. Do not proceed until verified."

**Step 03: The `/end-session` Skill**
> "Summarize the technical decisions of this session. Save result to `memory/project-x/sessions/[date]-[goal].md`."
</div>
</Section>

<Section>
<div slot="insight">
## 4. The Graduation Loop
The system improves by identifying local fixes with universal utility and promoting them to permanent standards.
</div>

<div slot="technical">
**Distillation Skill (/distill):**
> "Review last 7 days of logs. Identify 3 recurring patterns. Move to `global/lessons/`. Update project bootloader."
</div>
</Section>

<Section>
<div slot="insight">
## Result
A self-curating system. The LLM inherits its environment automatically, and the architecture absorbs new standards through a verified pattern of reuse.
</div>
</Section>
