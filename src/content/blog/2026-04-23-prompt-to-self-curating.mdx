## Principle
Prompting is a fallback. Reliability is achieved by moving from adjective-heavy instructions to a persistent **system** that surrounds the model. 

## Technical Requirements
To move beyond brittle prompts, the environment must satisfy three functional requirements:
1. **Persistence**: Context must survive across independent work sessions.
2. **Retrieval**: The model must prioritize local project data over its general training.
3. **Curation**: The system must graduate project fixes into permanent standards.
</div>
</Section>

<Section>
<div slot="insight">
## Memory: The Foundation
The primary requirement—Persistence—is implemented as a **Memory** directory. This provides the physical structure for short-term activity and long-term logic.
</div>

<div slot="technical">
**Folder Structure**
```text
memory/
├── state.json (global metadata)
├── global/
│   └── lessons/ (universal)
└── project-x/
    ├── sessions/ (activity logs)
    └── lessons/ (local standards)
```
</div>
</Section>

<Section>
<div slot="insight">
## System Rules
System Rules define the operating environment inside the **LLM**. A root file (e.g., `CLAUDE.md`, `.cursorrules`) enforces **Automatic Initialization** and triggers maintenance based on the state in Memory.
</div>

<div slot="technical">
**System Rules (e.g. `CLAUDE.md`)**
<div class="bootloader-snippet">
```markdown
# Project Standards
- **Init:** On boot, read `memory/state.json` and project context.
- **Curation:** If `last_distill` > 7 days, execute `/distill` before work.
- **Role:** Architect directs; LLM executes.
- **Log:** Use `/end-session` to persist results.
```
</div>
</div>
</Section>

<Section>
<div slot="insight">
## The Daily Workflow
Execution follows a deterministic cycle triggered by the Memory state:

1.  **Initialize**: LLM synchronizes with root state and context.
2.  **Curation**: LLM graduates rules if the system is stale.
3.  **Execute**: Architect directs; LLM implements.
4.  **Capture**: Run `/end-session` to persist the technical log.
</div>

<div slot="technical">
**The `/end-session` Skill**
> "Summarize technical decisions. Save to `memory/project-x/sessions/[date]-[goal].md`."

**The Graduation Skill (/distill)**
> "Identify recurring patterns. Move to `global/lessons/`. Update `memory/state.json`."
</div>
</Section>

A self-running architecture. The system monitors its own health from the highest level, ensuring standards never stagnate.
