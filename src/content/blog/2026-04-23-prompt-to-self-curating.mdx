<Section>
<div slot="insight">
## Principle
Prompting is a fallback. Reliability is achieved by moving from adjective-heavy instructions to a persistent **system** that surrounds the model. 

## The Requirement
A system requires a structured environment for context to persist across independent work sessions. This is implemented as a **Memory** directory on the local file system.
</div>
</Section>

<Section>
<div slot="insight">
## 1. Memory Structure
Memory is divided into two distinct layers:
- **Sessions**: Short-term logs of activity (what was tried, what failed).
- **Lessons**: Long-term engineering standards (distilled logic).
</div>

<div slot="technical">
```text
memory/
├── global/
│   └── lessons/ (universal)
└── project-x/
    ├── sessions/ (logs)
    └── lessons/ (local rules)
```
</div>
</Section>

<Section>
<div slot="insight">
## 2. The Bootloader
The Bootloader defines the operating environment inside the **LLM**. A root file (e.g., `CLAUDE.md`, `.cursorrules`) enforces retrieval and establishes the architect/worker relationship.
</div>

<div slot="technical">
**Bootloader (Project Root)**
<div class="bootloader-snippet">
```markdown
# Project Standards
- **Role:** Architect directs; LLM executes.
- **Recall:** Search `memory/` for logic.
- **Log:** Record outcome to `sessions/`.
- **Distill:** Propose rules for `lessons/`.
```
</div>
</div>
</Section>

<Section>
<div slot="insight">
## 3. The Graduation Loop
The system improves by identifying local fixes with universal utility and promoting them to permanent standards.
</div>

<div slot="technical">
| Step | Action |
|---|---|
| **Detection** | Identify recurring friction. |
| **Local Fix** | Add a rule to `lessons/`. |
| **Graduation** | Move to `global/lessons/`. |
</div>
</Section>

<Section>
<div slot="insight">
## Result
A self-curating system. The LLM inherits its environment automatically, and the architecture absorbs new standards through a verified pattern of reuse.
</div>

<div slot="technical">
**Distillation Skill:**
> "Review last 7 days of logs. Identify 3 recurring patterns. Move to `global/lessons/`. Update project bootloader."
</div>
</Section>
