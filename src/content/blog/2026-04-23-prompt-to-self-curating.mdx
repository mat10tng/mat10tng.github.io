## The Answer
**You can eliminate repetitive prompting and force your technical knowledge to compound across your entire portfolio by establishing a persistent, project-agnostic memory system.**

Reliability comes from an architecture where every session belongs to a specific project. By using a unified folder structure, you ensure that local fixes in one project can graduate into global standards for all future work.
</div>
</Section>

<Section>
<div slot="insight">
## 1. Portfolio Architecture
Compounding requires a strict directory hierarchy. This ensures the model has a deterministic location to store and retrieve history for any project in your portfolio.
</div>

<div slot="technical">
**Unified Memory Structure**
```text
memory/
├── state.json (global health)
├── global/
│   └── lessons/ (portfolio standards)
└── project-x/
    ├── sessions/ (daily logs)
    └── lessons/ (project fixes)
```

**Scaffold (Bash)**
```bash
mkdir -p memory/global/lessons
```
</div>
</Section>

<Section>
<div slot="insight">
## 2. The Foundation Skill (/init-project)
Every session must belong to a project. The `/init-project` skill is the first mandatory action for any new codebase, ensuring it is correctly registered within your Memory system.
</div>

<div slot="technical">
**Skill: `/init-project`**
> "Scaffold the memory structure for this new project: `memory/[project-name]/sessions/` and `memory/[project-name]/lessons/`. Initialize `memory/state.json` if this is the first project."
</div>
</Section>

<Section>
<div slot="insight">
## 3. The Operational Skills
Once initialized, the system uses two skills to capture and graduate logic.

**Capture**: Persists session-specific technical decisions.
**Graduation**: Identifies rules with portfolio-wide utility.
</div>

<div slot="technical">
**Skill: `/end-session`**
> "Summarize the technical decisions of this session. Save to `memory/[project-name]/sessions/[date]-[goal].md`."

**Skill: `/distill`**
> "Review logs from `[project-name]`. Identify rules that should graduate to `global/lessons/` to benefit the entire portfolio."
</div>
</Section>

<Section>
<div slot="insight">
## 4. System Rules
System Rules live inside the LLM. They enforce **Automatic Initialization**—forcing the model to identify the current project and synchronize with its specific Memory before work begins.
</div>

<div slot="technical">
**Project System Rules**
```markdown
# Project Standards
- **Init:** On boot, identify current project and read `memory/[project-name]/`.
- **Curation:** If `last_distill` > 7 days, execute `/distill`.
- **Role:** Architect directs; LLM executes.
- **Skills:** [/init-project, /end-session, /distill]
```
</div>
</Section>

<Section>
<div slot="insight">
## Result
A self-curating portfolio. The LLM inherits its environment automatically, and your engineering standards grow across every project you touch.
