<div slot="insight">
## Principle
Prompting is a fallback. Reliability is achieved by moving from adjective-heavy instructions to a persistent **system** that surrounds the model. 

## The Requirement
A system requires a structured environment for context to persist across independent work sessions. This is implemented as a **Memory** directory on the local file system.
</div>
</Section>

<Section>
<div slot="insight">
## 1. Memory Structure
Memory is divided into two distinct layers:
- **Sessions**: Short-term logs of activity (what was tried, what failed).
- **Lessons**: Long-term engineering standards (distilled logic).
- **State**: Persistent system metadata (e.g. `last_distill_date`).
</div>

<div slot="technical">
```text
memory/
├── global/
│   └── lessons/
└── project-x/
    ├── sessions/
    ├── lessons/
    └── state.json (system tracking)
```
</div>
</Section>

<Section>
<div slot="insight">
## 2. The Bootloader
The Bootloader defines the operating environment inside the **LLM**. It enforces **Automatic Initialization** and **Self-Curation** based on the current state in Memory.
</div>

<div slot="technical">
**Bootloader (Project Root)**
<div class="bootloader-snippet">
```markdown
# Project Standards
- **Init:** On boot, read `memory/` and `state.json`.
- **Curation:** If `last_distill` > 7 days, execute `/distill` before work.
- **Role:** Architect directs; LLM executes.
- **Log:** Use `/end-session` to persist results.
```
</div>
</div>
</Section>

<Section>
<div slot="insight">
## 3. The Daily Workflow
The system maintains itself. Curation is not a manual choice; it is triggered automatically by the project's state data.

1.  **Initialize**: On boot, the LLM reads state and context.
2.  **Curation**: If the system is stale, the LLM graduates rules immediately.
3.  **Execute**: Architect directs goals; LLM implements.
4.  **Capture**: Before closing, run `/end-session`.
</div>

<div slot="technical">
**The `/distill` Logic**
> "Identify 3 recurring patterns from recent logs. Move to `global/lessons/`. Update `last_distill` date in `state.json`."
</div>
</Section>

<Section>
<div slot="insight">
## Result
A self-running architecture. The system monitors its own health and enforces its own maintenance schedule, ensuring standards never stagnate.
</div>
</Section>
