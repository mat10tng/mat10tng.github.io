## The Answer
**You can eliminate repetitive prompting by establishing a persistent memory system.** 

Reliability comes from an architecture that surrounds the model, not the adjectives inside the prompt. By moving constraints from your chat box to a structured file system, you ensure every session inherits the correct technical context automatically.
</div>
</Section>

<Section>
<div slot="insight">
## 1. Functional Requirements
To move beyond brittle prompts, the environment must satisfy three logic gates:
1. **Persistence**: Context must survive across independent work sessions.
2. **Retrieval**: The model must prioritize local rules over its general training.
3. **Growth**: The system must automatically turn session fixes into permanent standards.
</div>

<div slot="technical">
| Requirement | Implementation |
|---|---|
| **Persistence** | Local Memory directory |
| **Retrieval** | Mandatory System Rules |
| **Growth** | Graduation of session logs |
</div>
</Section>

<Section>
<div slot="insight">
## 2. Memory Structure
The foundation is a deterministic location for project history. This ensures you never have to prompt the model with your basic stack or file paths more than once.
</div>

<div slot="technical">
**Folder Structure**
```text
memory/
├── state.json (metadata)
├── global/
│   └── lessons/ (universal)
└── project-x/
    ├── sessions/ (logs)
    └── lessons/ (local)
```

**Scaffold (Bash)**
```bash
mkdir -p memory/global/lessons memory/project-x/{sessions,lessons}
```
</div>
</Section>

<Section>
<div slot="insight">
## 3. System Rules
System Rules live inside the LLM (via `CLAUDE.md` or `.cursorrules`). They enforce **Automatic Initialization**—forcing the model to synchronize with Memory before accepting goals.
</div>

<div slot="technical">
**System Rules**
```markdown
# Project Standards
- **Init:** On boot, read `memory/state.json`.
- **Curation:** If `last_distill` > 7 days, execute `/distill`.
- **Role:** Architect directs; LLM executes.
- **Skills:** [/init-project, /end-session, /distill]
```
</div>
</Section>

<Section>
<div slot="insight">
## 4. Operational Skills
Skills are the "How-To" for the model. 

**Capture**: Summarizes the session outcomes.
**Curation**: Identifies recurring patterns and promotes them to permanent standards.
</div>

<div slot="technical">
**Skill: `/end-session`**
> "Summarize technical decisions. Save to `memory/project-x/sessions/[date]-[goal].md`."

**Skill: `/distill`**
> "Identify 3 recurring patterns from recent logs. Move to `global/lessons/`. Update `memory/state.json`."
</div>
</Section>
