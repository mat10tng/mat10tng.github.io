<div slot="insight">
**ELI5:** Instead of trying to write the perfect prompt every time, we build a simple folder system (Memory) that the AI reads before it starts working. This way, the AI "remembers" your rules and previous work automatically.

## The Goal
Moving from brittle, adjective-heavy instructions to a persistent **system** that surrounds the model. 

## Requirements
To move beyond manual prompts, the system needs:
1. **Memory**: Context must survive across different chats.
2. **Focus**: The model must prioritize your local rules over its general training.
3. **Growth**: The system must learn from fixes and turn them into permanent rules.
</div>

<div slot="technical">
| Need | Solution |
|---|---|
| **Memory** | Local folders |
| **Focus** | Mandatory System Rules |
| **Growth** | Graduation of logs |
</div>
</Section>

<Section>
<div slot="insight">
## 1. Memory: The Foundation
Memory provides the folders for storing short-term activity and long-term rules. This ensures context survives even if you start a fresh chat.
</div>

<div slot="technical">
**Folder Structure**
```text
memory/
├── state.json (metadata)
├── global/
│   └── lessons/ (universal)
└── project-x/
    ├── sessions/ (activity logs)
    └── lessons/ (local standards)
```

**Setup (Bash)**
```bash
mkdir -p memory/global/lessons memory/project-x/{sessions,lessons}
```
</div>
</Section>

<Section>
<div slot="insight">
## 2. The Skills
Skills are simple instructions that teach the model how to use its Memory.

### Capture (`/end-session`)
Summarizes the work and saves it to the right folder.

### Curation (`/distill`)
The "learning" part. The model looks at recent logs and creates permanent rules in the `lessons/` folder.
</div>

<div slot="technical">
**Skill: `/init-project`**
> "Scaffold the Memory structure for a new project: `memory/name/{sessions,lessons}`."

**Skill: `/end-session`**
> "Summarize technical decisions. Save to `memory/project-x/sessions/[date]-[goal].md`."

**Skill: `/distill`**
> "Identify 3 recurring patterns from recent logs. Move to `global/lessons/`. Update `memory/state.json`."
</div>
</Section>

<Section>
<div slot="insight">
## 3. System Rules
These rules live inside the **LLM** (via `CLAUDE.md` or `.cursorrules`). They force the model to initialize itself and follow your maintenance schedule automatically.
</div>

<div slot="technical">
**System Rules (e.g. `CLAUDE.md`)**
```markdown
# Project Standards
- **Init:** On boot, read `memory/state.json`.
- **Curation:** If `last_distill` > 7 days, execute `/distill`.
- **Role:** Architect directs; LLM executes.
- **Log:** Use `/end-session` to persist results.
- **Skills:** [/init-project, /end-session, /distill]
```
</div>
</Section>

<Section>
<div slot="insight">
## The Daily Loop
1. **Boot**: The model reads your state and context automatically.
2. **Learn**: If the system is stale, the model graduations rules first.
3. **Work**: You provide goals; the model implements them within your boundaries.
4. **Save**: You run `/end-session` to save the learning.
</div>
</Section>
