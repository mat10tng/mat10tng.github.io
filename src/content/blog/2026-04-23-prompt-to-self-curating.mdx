<Section>
<div slot="insight">
## Principle
Prompting is a fallback. Reliability is achieved by moving from adjective-heavy instructions to a persistent **system** that surrounds the model. 

## The Requirement
A system requires a structured environment for context to persist across independent work sessions. This is implemented as a **Memory** directory on the local file system.
</div>
</Section>

<Section>
<div slot="insight">
## 1. Memory Structure
Memory is divided into two distinct layers:
- **Sessions**: Short-term logs of activity (what was tried, what failed).
- **Lessons**: Long-term engineering standards (distilled logic).
</div>

<div slot="technical">
```text
memory/
├── global/
│   └── lessons/ (universal)
└── project-x/
    ├── sessions/ (logs)
    └── lessons/ (local rules)
```
</div>
</Section>

<Section>
<div slot="insight">
## 2. The Bootloader
The Bootloader defines the operating environment inside the **LLM**. A root file (e.g., `CLAUDE.md`, `.cursorrules`) establishes the architect/LLM relationship and enforces **automatic initialization**.
</div>

<div slot="technical">
**Bootloader (Project Root)**
<div class="bootloader-snippet">
```markdown
# Project Standards
- **Init:** Before any work, read `memory/` to synchronize context.
- **Role:** Architect directs; LLM executes.
- **Rules:** State approach before code.
- **Log:** Use `/end-session` to persist results.
```
</div>
</div>
</Section>

<Section>
<div slot="insight">
## 3. The Daily Workflow
The system runs by default. Initialization is not an option; it is the first mandatory action of every session.

1.  **Initialize**: On boot, the LLM reads the vault to synchronize with current standards.
2.  **Architect**: The Architect provides the goal; the LLM implements logic within system boundaries.
3.  **Capture**: Before closing, run `/end-session` to summarize and save the technical log.
</div>

<div slot="technical">
**The `/end-session` Skill**
> "Summarize the technical decisions of this session. Save result to `memory/project-x/sessions/[date]-[goal].md`."

**The Graduation Skill (/distill)**
> "Identify 3 recurring patterns from recent logs. Move to `global/lessons/`."
</div>
</Section>

<Section>
<div slot="insight">
## Result
A self-curating system that starts with context. The LLM inherits its environment automatically, ensuring that every session begins with the project's total history.
</div>
</Section>
