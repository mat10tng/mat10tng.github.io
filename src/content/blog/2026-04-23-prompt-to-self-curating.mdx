## 1. Memory: The Foundation
Memory provides the physical structure for short-term activity and long-term engineering standards. It satisfies the core requirement of **Persistence**—ensuring context survives across independent work sessions.
</div>

<div slot="technical">
**Folder Structure**
```text
memory/
├── state.json (metadata)
├── global/
│   └── lessons/ (universal)
└── project-x/
    ├── sessions/ (logs)
    └── lessons/ (local)
```
</div>
</Section>

<Section>
<div slot="insight">
## 2. System Rules
System Rules live within the **LLM** and define its operating environment. A root file established in the project (e.g., `CLAUDE.md`, `.cursorrules`) enforces **Automatic Initialization** and triggers maintenance based on the state in Memory.
</div>

<div slot="technical">
**System Rules (e.g. `CLAUDE.md`)**
<div class="bootloader-snippet">
```markdown
# Project Standards
- **Init:** On boot, read `memory/state.json` and project context.
- **Curation:** If `last_distill` > 7 days, execute `/distill` before work.
- **Role:** Architect directs; LLM executes.
- **Log:** Use `/end-session` to persist results.
```
</div>
</div>

<Section>
<div slot="insight">
## 3. The Daily Workflow
Execution follows a deterministic cycle. The result is a system that monitors its own health, ensuring standards never stagnate.

1.  **Initialize**: LLM synchronizes with state and context.
2.  **Curation**: LLM graduates rules if the system is stale.
3.  **Execute**: Architect directs; LLM implements.
4.  **Capture**: Run `/end-session` to persist the technical log.
</div>

<div slot="technical">
**The `/end-session` Skill**
> "Summarize technical decisions. Save to `memory/project-x/sessions/[date]-[goal].md`."

**The Graduation Skill (/distill)**
> "Identify recurring patterns. Move to `global/lessons/`. Update `memory/state.json`."
</div>
