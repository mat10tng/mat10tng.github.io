<div slot="insight">
## Principle
Prompting is a fallback. Reliability is achieved by moving from adjective-heavy instructions to a persistent **system** that surrounds the model. 

## The Requirement
A system requires a structured environment for context to persist across independent work sessions. This is implemented as a **Memory** directory on the local file system.
</div>
</Section>

<Section>
<div slot="insight">
## 1. Memory Structure
Memory is divided into two distinct layers:
- **Sessions**: Short-term logs of activity (what was tried, what failed).
- **Lessons**: Long-term engineering standards (distilled logic).
- **System State**: The `state.json` file lives at the root to track global health and maintenance schedules.
</div>

<div slot="technical">
```text

    тФЬтФАтФА sessions/
    тФФтФАтФА lessons/
```
</div>
</Section>

<Section>
<div slot="insight">
## 2. System Rules
System Rules define the operating environment inside the **LLM**. A root file established in the project (e.g., `CLAUDE.md`, `.cursorrules`) enforces **Automatic Initialization** and **Self-Curation** based on the global state in Memory.
</div>

<div slot="technical">
**System Rules (e.g. `CLAUDE.md`)**
<div class="bootloader-snippet">
```markdown
# Project Standards
- **Init:** On boot, read `memory/state.json` and project context.
- **Curation:** If `last_distill` > 7 days, execute `/distill` before work.
- **Role:** Architect directs; LLM executes.
- **Log:** Use `/end-session` to persist results.
```
</div>
</div>
</Section>

<Section>
<div slot="insight">
## 3. The Daily Workflow
The system maintains itself. Curation is not a manual choice; it is triggered automatically by the global state data.

1.  **Initialize**: On boot, the LLM reads the root state and project context.
2.  **Curation**: If the system is stale, the LLM graduates rules immediately.
3.  **Execute**: Architect directs goals; LLM implements.
4.  **Capture**: Before closing, run `/end-session`.
</div>

<div slot="technical">
**The `/distill` Logic**
> "Identify 3 recurring patterns from recent logs. Move to `global/lessons/`. Update `last_distill` date in `memory/state.json`."
</div>
</Section>

<div slot="insight">
## Result
A self-running architecture. The system monitors its own health from the highest level, ensuring standards never stagnate across any project.
