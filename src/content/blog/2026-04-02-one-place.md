at some point I had rules in config files, in memory files, in the notes system, in random docs. the same rule existed in three places, and none of the versions quite matched.

the problem wasn't the rules themselves — it was that the system didn't know which version was current. a new conversation loads one file, finds version A of a rule. searches the notes, finds version B. reads a memory file, finds version C. which one wins? depends on which got loaded first. same starting conditions, different behavior, no obvious reason why.

I only caught this because I pulled up a week of conversations and compared the rules each one was following. contradictions everywhere.

## the structure that works

one place is the source of truth — the notes system where everything is searchable and versioned. everything else becomes a thin pointer:

```
config file (loaded every session):
  → "load rules from the notes system before doing anything"
  → 5-6 hard-stop rules that can't wait for a search
  → nothing else

memory files:
  → pointers to notes, never content

the notes system:
  → all rules, all decisions, all lessons
  → searchable, versioned, can be compacted and archived
```

## what qualifies as a "hard-stop rule" for the config file

a rule only belongs in the config if ALL of these are true:
1. it applies to every single conversation regardless of task
2. violating it before the notes load would cause real damage
3. it can't be discovered fast enough through a search

examples that qualify: "never run the command that destroys the database." "always search the notes before starting work."

examples that don't qualify: communication preferences, planning workflows, debugging approaches. those live in the notes — they're discovered when relevant, not forced into every conversation's context.

## why this matters

duplication feels organized. "I'll put it here AND there, just to be safe." but duplication creates drift, and drift creates contradictions, and contradictions create unpredictable behavior. one source of truth, everything else points to it.
