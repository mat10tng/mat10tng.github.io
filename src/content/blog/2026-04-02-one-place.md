title: "one place, not twelve"
description: "rules were scattered everywhere. one decision fixed it."
pubDate: 2026-04-02
loop: "improve"
---

at some point I had rules in config files, in memory files, in the notes, in random docs. same rule, three versions, none quite matching.

the problem wasn't the rules — it was that nobody knew which version was current. a new conversation loads one file, finds one version. searches the notes, finds another. reads a memory file, finds a third. which one wins?

I only caught this because I reviewed a week of conversations and noticed contradictions. the system was following different versions of the same rule depending on which file it read first. same input, different behavior, no obvious reason why.

the fix was simple: one place for everything. the notes system is the single source of truth. config files became thin pointers that say "load the rules from here." no duplication. no drift.

the hard part wasn't the architecture. it was admitting that having rules in three places felt organized but was actually chaos.
