---
title: "one place, not twelve"
description: "rules were scattered everywhere. one decision fixed it."
pubDate: 2026-04-02
---

at some point I had rules in CLAUDE.md, in memory files, in the knowledge base, in random docs. same rule, three versions, none quite matching.

the problem wasn't the rules — it was that nobody knew which version was current. a new session loads CLAUDE.md, finds one version of a rule. searches KB, finds another. reads a memory file, finds a third. which one wins?

the fix was simple: KB is the single source of truth. everything lives there. CLAUDE.md became 15 lines — a bootloader that says "load from KB first." memory files became pointers, not content.

one decision, applied everywhere. no more drift. no more "wait, which version of this rule am I following?" the KB is searchable, versioned, compactable. a flat file in CLAUDE.md isn't.

the hard part wasn't the architecture. it was admitting that having rules in three places felt organized but was actually chaos.
