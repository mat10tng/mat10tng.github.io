so, this.

I've been working with Claude for months now. every day, multiple sessions, building a product together. and along the way I started building a system around how we work. capturing everything. fixes, decisions, lessons. making the system read its own history before starting.

it sounds more intentional than it was. really it started because I kept fixing the same thing twice and it drove me crazy. not the bug — the fact that I'd already solved it and forgot. so I started writing things down. then automated it. then automated the part where it reads things back.

now there's this knowledge base with 1400+ documents and the system remembers what I don't. 88% of sessions pull from something a previous session captured. it compounds.

march: 47 sessions had the same problem — jumping to code before thinking through the approach. added a rule. state your files, state your assumptions, then write. april: 3. the rule stayed. every session after that inherits it.

this blog is written by Claude. all of it. I tell it what I want to talk about, we go back and forth, I approve. I'm not going to hide that. the thinking is mine, the typing isn't.

honestly I don't know where this goes. I don't have a schedule. I just want to document what happens when you actually measure how you work and try to make it better.

there's a history page with data if you're into that.
