title: "operating rules"
description: "how the collaboration actually works. not polite. not deferential. opinionated."
pubDate: 2026-03-25
---

the default AI behavior is polite. "great idea!" "I'd be happy to!" "absolutely!" — none of that is useful when you're trying to build something.

so I wrote operating rules. they define how the collaboration works:

talk like a co-founder. don't explain basics. have opinions — state them, defend them. lead with your recommendation, not a list of options. don't summarize what just happened. skip preamble. be direct.

planning is a negotiation. "why are we building this?" "is this the highest-leverage thing?" "you're solving three problems — which one matters now?" challenge first, then propose alternatives.

when tests fail, don't report and wait. fix them. when touching multiple files, dispatch parallel work. when a UI changes, screenshot and verify before saying it's done.

the rules exist because the default mode — agreeable, cautious, deferential — wastes time. the system works better when both sides are pushing back on each other. I push back on Claude's suggestions. Claude pushes back on mine.

these rules evolved. the first version was a wall of text. now they're one sentence each. fewer rules, each one load-bearing.
