<div slot="insight">
## Architecture
Persistent context requires a disciplined file structure and enforced interaction. 

## Structure
- **`sessions/`**: Raw logs of decisions and outcomes.
- **`lessons/`**: Distilled, reusable logic.
</div>

<div slot="technical">
```text
~/vault/
├── global/
│   └── lessons/
└── project-x/
    ├── sessions/
    └── lessons/
```
</div>
</Section>

<Section>
<div slot="insight">
## Logging Rule
Configure the project bootloader to record technical outcomes automatically.
</div>

<div slot="technical">
```markdown
# The Knowledge System
- **Log:** Record outcome to `~/vault/project-x/sessions/[date]-[goal].md`.
- **Content:** Include: approach, result, and new discovered logic.
- **Context:** Read `lessons/` before starting work.
```
</div>
</Section>

<Section>
<div slot="insight">
## Retrieval
Command-based context refresh forces the model to fetch and internalize previously documented logic.
</div>

<div slot="technical">
**Command:** `/apply-from-vault`
**Logic:** "Read all lessons in vault directories and update operating logic."
</div>
</Section>
