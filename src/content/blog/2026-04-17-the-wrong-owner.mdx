## Principle
Reliability is achieved by removing probabilistic tasks from the LLM and transferring them to deterministic code.

## The Logic
When a task fails, ask: "Should the model be doing this at all?"
</div>

<div slot="technical">
| Task | Correct Owner |
|---|---|
| **Bookkeeping** | Code/Database |
| **Enforcement** | CI/Scripts |
| **Synthesis** | LLM |
</div>
</Section>

<Section>
<div slot="insight">
## Implementation: Uniqueness
Instead of instructing the model to "be careful" with identifiers, implement a validation function in the interface.
</div>

<div slot="technical">
```javascript
function validateId(id, set) {
    if (set.has(id)) throw Error("Duplicate identifier detected");
}
```
</div>
</Section>

<Section>
<div slot="insight">
## Result
Transferring the job to the surrounding system is more effective than refining the prompt instruction.
</div>
