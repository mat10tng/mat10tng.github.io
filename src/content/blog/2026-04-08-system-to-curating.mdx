## Principle
A static system degrades. Continuous improvement is achieved through a mechanism that identifies and graduates local fixes into universal utility.

## Graduation Path
1. **Detection**: Identify recurring friction in the system.
2. **Local Fix**: Implement project-level rules in `memory/project-x/lessons/`.
3. **Pattern**: Observe the fix solving issues in unrelated projects.
4. **Graduation**: Move the logic to global standards in `memory/global/lessons/`.
</div>

<div slot="technical">
| Step | Action |
|---|---|
| **Capture** | Log friction in a session file. |
| **Fix** | Add rule to project `lessons/`. |
| **Move** | Relocate to `global/lessons/`. |
</div>
</Section>

<Section>
<div slot="insight">
## Distillation Skill
Enable self-curation via periodic review. This process identifies high-utility logic and removes session-specific noise from Memory.
</div>

<div slot="technical">
**Command:** `/distill`
**Logic:** "Review recent logs. Identify 3 recurring patterns. Move to `global/lessons/`. Update project bootloader."
</div>
</Section>

<Section>
<div slot="insight">
## Result
The shift from a static system to a self-curating one. The system automatically absorbs new engineering standards into its long-term Memory through a verified pattern of reuse.
</div>
